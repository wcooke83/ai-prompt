ChatGPT "Copy code" Button Investigation

Expected HTML Structure (as of late 2024)

<!-- Code block container -->
<div class="contain-inline-size rounded-md border-[0.5px] border-token-border-medium relative bg-token-sidebar-surface-secondary my-4">

  <!-- Header with language and copy button -->
  <div class="flex items-center text-token-text-secondary px-4 py-2 text-xs font-sans justify-between rounded-t-md h-9 bg-token-sidebar-surface-primary dark:bg-token-main-surface-secondary">
    <span>python</span>

    <!-- Copy code button -->
    <button class="flex gap-1 items-center">
      <svg ... /> <!-- Copy/check icon -->
      "Copy code" or "Copied!"
    </button>
  </div>

  <!-- Code content -->
  <div class="overflow-y-auto p-4">
    <code class="!whitespace-pre hljs language-python">
      ... code content ...
    </code>
  </div>
</div>

Key Observations

  • The "Copy code" button does NOT have a data-testid attribute
  • It's inside a header div that's a sibling to the code content div
  • Button text changes from "Copy code" to "Copied!" after clicking
  • Uses modern Clipboard API (navigator.clipboard.writeText())

Why Programmatic Click Might Fail

  1. Trusted Events: Browsers may require user-initiated (trusted) events for clipboard access
  2. React Event Handling: React uses synthetic events; DOM-level events may not trigger handlers
  3. Focus/Visibility: Clipboard API may require document focus
  4. Permissions: Extension content scripts may lack clipboard-write permission

Reliable Selectors

// Find all code blocks
document.querySelectorAll('pre code')

// Find code block containers (each has a header with copy button)
document.querySelectorAll('div.contain-inline-size.rounded-md')

// Find Copy code buttons by text content
Array.from(document.querySelectorAll('button')).filter(b => b.textContent.includes('Copy code'))

// Find by structure: button in code block header
document.querySelectorAll('div.contain-inline-size button.flex.gap-1')

Alternative: Direct Code Extraction (More Reliable)

// Instead of clicking the button, extract code directly:

function extractCodeFromBlock(codeBlockContainer) {
  const codeElement = codeBlockContainer.querySelector('code');
  return codeElement ? codeElement.textContent : null;
}

// Get all code blocks
const codeBlocks = document.querySelectorAll('pre code');
codeBlocks.forEach((code, i) => {
  console.log(`Code block ${i}:`, code.textContent);
});

The Copy Button's Likely Implementation

// ChatGPT's React component likely does something like:

const handleCopy = async () => {
  try {
    await navigator.clipboard.writeText(codeContent);
    setButtonText('Copied!');
    setTimeout(() => setButtonText('Copy code'), 2000);
  } catch (err) {
    // Fallback to document.execCommand
    const textarea = document.createElement('textarea');
    textarea.value = codeContent;
    document.body.appendChild(textarea);
    textarea.select();
    document.execCommand('copy');
    document.body.removeChild(textarea);
  }
};

Extension Fix Recommendations

  1. Don't click the button - Just extract the text directly from <code> elements
  2. If button click is required:
    • Use element.click() directly, not synthetic events
    • Ensure document has focus first
    • Grant clipboard permissions in manifest.json
  3. For response copying: Use button[data-testid="copy-turn-action-button"] which is the main "Copy" button for the entire response
